A. Comparing similarity scores
Out of all the algorithms the Exact Position Match algorithm is the weakest because it only compares characters at the same index and fails when sequences are shifted or contain insertions or deletions. The longest common substring is better but still sensitive to mutations/changes. The longest common subsequence allowed for better comparison since it allows gaps, making it better suited for sequences where there can be insertions and deletions. The Levenshtein distance is the best out of all the algorithms because it models mutation through insertions, deletions, and it provides a true edit cost between sequences. 



B. Measuring performance
The Exact Position Match algorithm is the most efficient, running in linear time O(n)with minimal memory usage, since it only requires a single pass through both sequences. while, the Longest Common Substring and Longest Common Subsequence algorithms rely on dynamic programming, giving them time and space complexity of O(m×n). Among these, substring is typically the fastest in practice due to simpler computations, while LCS is slightly slower because it must compare multiple possibilities at each step. Levenshtein distance tends to be the slowest of the three because it performs more operations per cell (insert, delete, substitute). If implemented recursively without optimization, Levenshtein becomes extremely inefficient (exponential time), but with dynamic programming, all three remain practical for moderate sequence lengths.
